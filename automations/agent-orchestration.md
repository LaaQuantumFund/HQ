description: 複数エージェントの協調作業・タスク分割・進捗管理を自動調整する
エージェントオーケストレーションスキル

適用タイミング
- ユーザーの指示が複数エージェントの領域にまたがる場合
- 「○○をリリースして」等の包括的な指示があった場合
- エージェント間でデータの受け渡しが必要な場合

オーケストレーションルール

タスク分解
1. ユーザーの指示を単一エージェントで完結するサブタスクに分割する
2. サブタスク間の依存関係を明確にする（並列可能 / 直列必須）
3. 各サブタスクに担当エージェントを割り当てる
4. 分割結果をユーザーに提示し承認を得る

担当エージェントの判定基準
| 作業内容 | 担当 |
|----------|------|
| コード実装・修正・デプロイ | developer |
| コンテンツ作成・SNS・SEO | marketer |
| 収支記録・税務・法人事務 | accountant |
| トレード分析・Bot ロジック設計 | trader |
| 調整・整理・横断管理 | secretary |

依存関係の処理
- 直列タスク: 前工程の成果物を明示的にファイルまたは Linear issue コメントで引き継ぐ
- 並列タスク: 各エージェントが独立して作業し、secretary が成果物を統合
- 合流ポイント: 全サブタスク完了後に secretary がサマリーを作成し社長に報告

実行パターン例

パターン A: note 記事の公開
1. [marketer] Obsidian から素材収集 → 記事下書き作成
2. [社長] 下書き承認
3. [developer] note-generator で記事をフォーマット・公開準備
4. [marketer] SNS 告知文を作成
5. [社長] 最終承認
6. [developer] 公開実行 + SNS 自動投稿トリガー
7. [secretary] 完了記録・Linear 更新
パターン B: 新ツールのリリース
8. [developer] 実装・テスト・ビルド
9. [marketer] LP コピー・スクリーンショット・告知文
10. [accountant] 価格設定・決済フロー確認
   ↑ 2と3は並列実行可能
11. [社長] 最終レビュー・承認
12. [developer] デプロイ
13. [marketer] SNS + note で告知
14. [secretary] 完了記録・dashboard 更新
パターン C: 月次レビュー
15. [accountant] 月次収支集計
16. [trader] 月次トレード分析
17. [marketer] 月次コンテンツパフォーマンス集計
   ↑ 1〜3は並列実行可能
18. [secretary] 全レポートを統合し社長に提出
19. [社長] レビュー・意思決定
20. [secretary] 決定事項を Linear に反映
進捗管理
- 各サブタスクの開始・完了を dashboard/agent-log.md に記録
- ブロッカーが発生したら即座に社長に報告（mobile-ops スキルの緊急度表示を使用）
- 想定より遅延している場合は原因と対策を提示

制約
- 4つ以上のサブタスクに分割する場合は必ず社長の承認を得る
- エージェント間の直接指示は行わない（全て Linear issue またはファイル経由）
- 各エージェントの制約（金銭操作禁止等）はオーケストレーション時も厳守
