AI社員システム構築

構想（Aaron Sneed「The Council」モデル参考）

一人法人の全業務を AI エージェントで構成する。Phase 3 では既に定義済みの5体のエージェントを本格稼働させる。

エージェント稼働計画

優先度1: secretary（AI秘書 / Chief of Staff）
稼働目標: 5月 Week 1
理由: 他エージェントの調整・レポート集約を担うため最初に安定させる
稼働確認項目:
- [ ] /daily-briefing が毎朝自動で Slack に届く
- [ ] Linear のイシュー集計が正確
- [ ] agent-log.md への記録が自動化されている
- [ ] 複数エージェントの成果物を統合したレポートを生成できる

優先度2: developer（開発リード）
稼働目標: 5月 Week 1〜2
理由: HP 公開・CI/CD 構築を担う
稼働確認項目:
- [ ] HP のコード生成・Vercel デプロイが完了
- [ ] GitHub Actions CI が全リポジトリで稼働
- [ ] PR 作成・レビューフローが機能
- [ ] 各プロジェクトの CLAUDE.md に従った開発ができる

優先度3: trader（トレード分析）
稼働目標: 5月 Week 2
理由: 4月のジャーナルデータが蓄積されており、統計分析の価値が高まる
稼働確認項目:
- [ ] vault のジャーナルデータを読み取り統計を生成できる
- [ ] 勝率・RR比・ドローダウン等の自動集計
- [ ] パターン検出と改善提案の生成
- [ ] dashboard への分析結果出力

優先度4: marketer（マーケティング）
稼働目標: 5月 Week 3
理由: HP 公開後に SNS 再始動を行う
稼働確認項目:
- [ ] vault のナレッジを基にした SNS 投稿ドラフト生成
- [ ] CEO 承認フロー（ドラフト → 承認 → 公開）が機能
- [ ] 最低1つの SNS アカウント（X, Instagram, note のいずれか）で投稿開始

優先度5: accountant（経理）
稼働目標: 5月 Week 3〜4
理由: 法人版 freee への移行完了後に稼働
稼働確認項目:
- [ ] financial-summary.md の月次自動更新
- [ ] 税務期限リマインダーの設定
- [ ] 法人の経費・売上の記録フローが確立

AI社員の管理原則
- 段階的に稼働させる（一度に全部動かさない）
- 各エージェントの稼働確認は CEO が直接テストする
- 問題が発生したら即座に該当エージェントを停止し、原因を調査
- 重要な判断には必ず CEO 承認を挟む（CLAUDE.md のコアポリシー参照）
- ハルシネーションリスクを常に意識し、出力の事実確認を怠らない

HP（ホームページ）公開

概要
- 法人の公式 Web サイトを ~/dev/hp/ で開発し Vercel にデプロイする
- 初期は最小構成（会社概要、事業内容、お問い合わせ）で公開
- developer エージェントが実装、marketer エージェントがコンテンツ提案

公開スケジュール
- 5月 Week 1: デザイン・構成決定（Laa Foundation で議論）
- 5月 Week 2: developer が実装・テスト
- 5月 Week 2〜3: CEO レビュー → 修正 → 公開
- 5月 Week 3〜: marketer が SEO 最適化・コンテンツ追加

SNS 再始動

対象アカウント

- 週2〜3投稿から開始（CEO 承認フロー付き）
- 1Password に全アカウントの認証情報を保管確認

退職手続き

スケジュール
- 4月末: 退職意向を上司に伝達済み（短期目標で達成）
- 5月前半: 正式な退職届提出
- 5月中: 業務引き継ぎ
- 6月末: 最終出社日

注意事項
- 健康保険: 退職後は法人の社会保険に切り替え（設立届出で対応）
- 年金: 厚生年金を法人で継続
- 住民税: 一括徴収 or 普通徴収の選択
- 失業保険: 法人代表は受給資格なし（自己都合退職＋即法人代表のため）
- 退職金: 有無を確認
- 有給消化: 残日数を確認し計画的に使用
