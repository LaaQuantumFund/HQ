# Security Policy

## 絶対禁止事項
- API キー・パスワード・シークレットをコード・ドキュメントに記載しない
- 送金・振込・決済の自動実行（社長が手動で行う）
- 外部サービスへの認証情報の平文送信
- 社長の個人情報（本名・住所・口座番号）の記載

## アクセス制御
- Brain への書き込み: COS のみ（読み取りのみ・書き込みは社長の専権）
- accounting/: CFO のみ（CEO 承認後）
- legal/: CLO のみ（CEO 依頼時）
- ~/dev/: CTO のみ

## シークレット管理
- 環境変数または .env ファイル（.gitignore 必須）
- MCP 認証情報は ~/.claude/ 配下の設定ファイルに記載

## インシデント対応
1. 検出 → 即座に社長へ Slack 通知
2. 影響範囲の特定
3. キーのローテーション
4. 原因分析と再発防止策
