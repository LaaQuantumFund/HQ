短期目標プラン：裁量トレード安定化

最終更新: 2026-03-31
期間: 2026年4月（4週間）
対応フェーズ: Phase 1 + Phase 2

ゴール定義

最終ゴール
裁量トレードで安定した利益を出し、貯金200万円超を達成し、退職意向を伝える。

達成基準
- 貯金残高200万円超（給与 + トレード利益の合算）
- トレードルールの遵守率 90% 以上（ジャーナル自己評価ベース）
- 週次・月次レビューの実施率 100%
- 4つの柱（マインドセット・習慣・環境・手法）が vault に文書化され、運用されている状態
- 退職意向を伝える決断ができる状態（精神的・経済的）

補足
月間勝率やリスクリワード比などの数値目標は、4月中のトレードデータが蓄積された段階で trader エージェントと Trade Mastery での評価を経て設定する。最初の2週間はデータ収集期間とする。

4つの柱

柱①：マインドセット（トレーダーとしての思考基盤）

目的
感情的なトレードを排除し、ルールベースの意思決定を定着させる。

錬成フロー
1. GenSpark で壁打ち（認知バイアス、感情パターン、メンタルモデルの議論）
2. 得られた洞察を vault/02_areas/mindset/ にノートとして保存
3. 実トレードで検証（ジャーナルに感情ログを記録）
4. Trade Mastery の週次レビューで上司（AI管理者）から評価を受ける
5. 改善点を再び GenSpark で深掘り → vault 更新

具体的な取り組み
- トレード前のマインドチェックリスト作成・運用（vault テンプレート）
- トレード後の感情ログ記入（vault/06_daily/ のジャーナル内）
- 認知バイアスリスト作成（vault/02_areas/mindset/cognitive-biases.md）
- 「やってはいけないリスト」の作成と壁への貼り出し
- 連敗時・大勝時のメンタルプロトコル策定

保存先
- vault/02_areas/mindset/ — マインドセット関連ノート群
- vault/02_areas/trading/ — ジャーナル内の感情ログ

柱④：手法（具体的なトレード手法の確立）

目的
明文化されたルールに基づいて一貫性のあるトレードを行う。

錬成フロー
1. GenSpark で手法の仮説を議論（エントリー条件、エグジット条件、時間帯、通貨ペア）
2. TradingView でバックテスト（チャート上で過去検証）
3. 結果を vault/02_areas/trading/backtest/ に記録
4. 十分なサンプルが貯まったら trader エージェントに統計分析を依頼
5. ルールブック更新（vault/02_areas/trading/rules/）
6. リアルトレードで検証
7. ジャーナルで実績追跡 → 週次レビューで改善サイクル

ルールブックの構成（vault/02_areas/trading/rules/）
- entry-rules.md — エントリー条件の定義
- exit-rules.md — 利確・損切り条件の定義
- risk-management.md — リスク管理ルール（1トレード1〜2%、最大同時ポジション数等）
- session-rules.md — トレードする時間帯・セッションの定義
- pair-selection.md — 通貨ペアの選定基準
- forbidden-patterns.md — 絶対にやらないパターンのリスト

手法改善サイクル
GenSpark壁打ち → 仮説 → バックテスト → vault記録
      ↑                                    ↓
   改善案 ← trader分析 ← ジャーナル蓄積 ← リアルトレード

失敗時のプロトコル

トレードルールを破った場合
1. ジャーナルに詳細を記録（何を破ったか、なぜ破ったか、感情は何だったか）
2. その日のトレードを停止する
3. Trade Mastery の日報で正直に報告 → AI 上司から評価を受ける
4. GenSpark で根本原因を壁打ち
5. 対策を vault/02_areas/mindset/ に追記
6. tasks/lessons.md に記録

大きな損失を出した場合
1. 即座にトレードを停止する（最低24時間）
2. ジャーナルに記録
3. GenSpark でメンタル回復の壁打ち
4. リスク管理ルールの見直し（vault/02_areas/trading/rules/risk-management.md）
5. Trade Mastery で復帰プランを相談

ルーティンが途切れた場合
1. 途切れた理由をジャーナルに記録
2. 翌日から再開する（途切れたことを責めない）
3. Trade Mastery の日報で報告
4. 3日以上連続で途切れた場合は GenSpark でルーティン自体を見直す
