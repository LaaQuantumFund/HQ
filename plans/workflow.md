## 1. 組織構成と運用体制

### 1.1 組織図

社長（あなた）
  └── CEO エージェント（Opus / Sonnet 併用）
        ├── COS（秘書 / Haiku）
        ├── CTO（技術統括 / Sonnet）
        ├── CMO（マーケティング / Sonnet）
        ├── CFO（経理・財務 / Haiku）
        ├── CIO（投資分析 / Sonnet）
        ├── CQO（品質管理 / Haiku）
        └── CLO（法務 / Haiku）

※ 日報・週報のレビューは Claude Project（外部）で実施。HQ エージェント組織の管轄外。

### 1.2 役割の関係

- **社長（あなた）**: 最終意思決定者。CEO に指示を出す。
- **CEO エージェント**: CxO チームを統括し、日次ブリーフィング・週次レポートを作成。Slack の #ceo チャンネルで報告。重要判断は Opus、ルーティンは Sonnet で実行。
- **各 CxO**: 担当領域のタスクを実行。CEO の指示で動く。

### 1.3 モデル選定方針

| エージェント | モデル | 理由 |
|---|---|---|
| CEO | Opus（判断時）/ Sonnet（ルーティン） | 統括・意思決定は Opus、日次報告は Sonnet |
| COS | Haiku | 秘書・事務処理。軽量で十分 |
| CTO | Sonnet | 技術設計・コードレビュー |
| CMO | Sonnet | コンテンツ企画・マーケ戦略 |
| CFO | Haiku | 定型経理処理 |
| CIO | Sonnet | 投資分析（定型分析は Sonnet で十分） |
| CQO | Haiku | チェックリストベースの品質確認 |
| CLO | Haiku | 定型法務チェック |

### 1.4 日報・週報の流れ

**CEO → あなた:**
- 毎日: /daily-briefing で事業サマリーを Slack #ceo に投稿
- 毎週金曜: 週次レポートを dashboards/weekly-report.md に作成・Slack #ceo に投稿

**あなたの日報に書くこと:**
- 今日やったこと（トレード成績、学習内容、開発進捗）
- ルール遵守率（トレードルールを守れたか）
- メンタル状態（1-10スケール）
- 明日の予定
- 気づき・反省

## 8. タスク管理（Linear）

### チーム構成

- **Trading** -- トレード関連タスク（学習、ルール策定、ジャーナル）
- **Business** -- 法人運営・事業全般タスク

### ワークフロー

Backlog → Todo → In Progress → Done → Canceled

### タスクの作り方

Claude Code から:
→「Linear に issue を作成して。チーム: Trading、タイトル: エントリールール v1 を策定、説明: GenSpark での壁打ち結果をもとに entry-rules.md を作成、優先度: High」

Slack から（Phase 2 以降）:
→ #ceo に「エントリールール策定のタスクを作って」と投稿 → CEO エージェントが Linear に issue 作成

### タスク管理のルール

- 1日の作業開始時に Linear を確認し、今日やるタスクを In Progress に移動
- 完了したら Done に変更
- /daily-briefing で残タスク数が自動報告される
- 週次レビューで Backlog の棚卸しを行う
