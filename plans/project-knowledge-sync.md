Claude Desktop Projects ナレッジ更新ルール

最終更新: 2026-03-31
対象: Trade Mastery / Laa Foundation の2プロジェクト

概要

Claude Desktop Projects にはナレッジベース機能がある。ドキュメントをアップロードすることで、そのプロジェクト内の全会話で Claude が参照できるようになる。

ただし、ナレッジベースはあくまで vault の「要約コピー」である。マスターデータは常に vault（/vault/）または hq（/hq/）に置く。Projects のナレッジが古くなった場合は vault から再エクスポートして差し替える。
vault（マスターデータ）
    │
    │ 要約・抜粋をエクスポート
    │
    ├──→ Trade Mastery ナレッジベース
    │
    └──→ Laa Foundation ナレッジベース
Trade Mastery のナレッジ構成

役割の再確認
Trade Mastery は「AI上司」として日報・週報の受理、Linear/GitHub データの突合、評価・フィードバックを行う。ナレッジベースには、AI上司が正確な評価を下すために必要な文脈を入れる。

登録しないもの
- 個別のトレードジャーナル（量が膨大。必要に応じて GitHub MCP で参照する）
- バックテストの生データ（要約のみ登録）
- GenSpark のチャット履歴（vault に転記済みの洞察のみ登録）
- hq のインフラ系ドキュメント（Trade Mastery の役割外）

要約の作成方法

基本方針
- ナレッジベースの容量には制限があるため、vault の内容をそのまま入れるのではなく要約版を作成する
- 要約は「AI上司 / AI参謀が判断を下すために最低限必要な情報」に絞る
- 詳細が必要な場合は、Projects 内の会話で「GitHub MCP で vault/○○/○○.md を確認して」と指示すれば原本を参照できる

要約テンプレート
[ドキュメント名]
ソース: [vault/hq のパス]
最終更新: YYYY-MM-DD

要点
（3〜5行で結論・現状を記載）

詳細
（必要に応じて構造化した内容。長くなりすぎない）

参照先
（原本のパス。詳細が必要な場合は GitHub MCP でこのパスを参照）

運用上の注意

ナレッジの鮮度
- 古いナレッジは AI の判断を誤らせる。特にトレードルールが改訂されたのにナレッジが旧版のままだと、Trade Mastery の評価がずれる
- 更新を忘れないよう、ルール改訂時は tasks/todo.md に「Trade Mastery ナレッジ更新」タスクを追加する

ナレッジの重複防止
- 同じ内容を Trade Mastery と Laa Foundation の両方に入れない
- 短期のトレード関連は Trade Mastery のみ
- 中長期の事業関連は Laa Foundation のみ
- 両方に関わる情報（例: 全体ロードマップ）は Laa Foundation に入れ、Trade Mastery では参照が必要な場合に会話内で言及する

MCP との使い分け
- ナレッジベース: 毎回の会話で自動的に読み込まれる。頻繁に参照する基本情報を置く
- Linear MCP: リアルタイムのタスク状態を取得する。ナレッジには入れない
- GitHub MCP: vault の詳細な内容を必要時に参照する。ナレッジには要約だけ入れ、詳細は MCP で取る
