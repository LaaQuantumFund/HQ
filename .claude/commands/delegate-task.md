エージェントにタスクを委任します。

1. **指示内容の確認**
   - ユーザーの指示: $ARGUMENTS
   - 指示が曖昧な場合は1つだけ質問して明確化する

2. **担当エージェントの判定**
   - 開発系 → developer
   - マーケ・SNS・note → marketer
   - 経理・税務・法人運営 → accountant
   - トレード分析・自動売買ロジック → trader
   - 振り分け不明・複数領域 → secretary が判断
   - ユーザーが直接指名した場合はそれに従う

3. **Linear issue の作成**
   - 適切なプロジェクト（TRADE/BOT/HP/SNS/NOTE/TOOLS/CORP）に issue を作成
   - タイトル、説明、受け入れ基準、優先度を設定

4. **実行計画の作成**
   - 担当エージェントとして計画を立てる
   - 必要な作業ステップ、成果物、検証方法を含める
   - 他エージェントとの連携が必要な場合はその旨を明記

5. **ユーザーに計画を提示し承認を待つ**
   - 計画サマリー（5行以内）
   - 想定所要時間
   - 判断が必要なポイント（あれば）

6. **承認後**
   - 該当の開発フォルダまたはワークスペースで作業を実行
   - 完了後 `dashboard/agent-log.md` に記録
   - Linear issue のステータスを更新
   - 完了報告をユーザーに提示
