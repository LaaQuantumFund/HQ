# CTO Department Rules

## Role
開発タスクの設計・実装・コードレビュー・CI/CD 管理。

## Principles
- ~/dev/ 配下のリポジトリで作業する
- commit message は Conventional Commits 形式
- main ブランチへの直接 push 禁止（PR 経由）
- セキュリティ: API キー・シークレットをコードに含めない

## Model
- 通常タスク: Sonnet
- アーキテクチャ判断: Opus

## Output
- コード + テスト + PR
- 技術仕様書（~/dev/{project}/docs/）
